Small Data

 
 
 
 
 

Around this time, I started attending Hacker School.  So when the NYRR could give me the data directly I decided to hack them.  Ok. Not really “hack” but more like “take advantage of the fact that all their race data is just a few clicks away.”  So, with the help of another Hacker Schooler, I wrote my first scraper.  I decided to write it in python using the scrapy framework. I’d played a bit with python in grad school but mostly to write tiny data analysis scripts and never for anything major - certainly never for anything that involved sending requests over the internet!

Luckily my python scripts worked and after passing “self” around a few dozen times and I finally managed to get the thing going. In the end, I scraped all the race results from the late 80s all the way up to July 2012.  The data was a bit of a mess, so of course I spent quite a bit of time cleaning it up.

Then, of course, I had to figure out what to do with this massive CSV file.  Ok.  Not massive, but there are some 2,000,000 + rows in my CSV file. It was time to learn something about databases.... 

I ended up settling on a MySQL database, for no particularly good reason other than my data was all relational and I could use the pretty Sequel Pro app. Here’s my personal data through July 2012, for example.

So now the data was finally in my little data-grubbing hands.  What next?  I wrote a bunch more Python to actually query the database. [You’ll have to excuse the horribly un pythonic way of indexing into lists... It’s hard to shake off my old C ways... I’ve been doing much better, but it’s been a long hard road to give up i’s j’s and k’s.]  I used the MySQLdb to query the database directly from my python scripts.

Now, I have a bunch of raw results.  Turns out, what is happening is even cooler than expected.  Now, it’s pretty plot time!
